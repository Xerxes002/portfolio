Introduction

In I.T. cloud computing has provided professionals additional flexibilty when building infrastructure. By using the cloud, there is a potential to reduce costs and create additional flexibility when compared to using on-site hardware. However, even though the machines are in the cloud, they are still suseptible to attacks. That's why it is still important to monitor these machines for security events.

In this project we will be using Microsoft Azure to create a network and configure a SIEM. This guide will walk through configuring, optimizing, and securing an I.T. environment in the cloud.

Creating a topology diagram

To start, it is important to create a topology diagram of the network help outline what devices will be placed within the cloud and how they will be connected to eachother. In my setup, I am using a dedicated computer for pfSense. However, I will be using Hyper-V to host several Windows machines and vSphere to host Security Onion.

Logging into Microsoft Azure

First we will use a Microsoft account to sign up for Azure. Azure provides a 30 day free trial along with $200 of credit for deploying resources. Once signed up you will see this dashboard.

Creating a resource group

Next, we will create a resource group. We will use a resource group to group all the resources we will be using in this lab. In the search bar at the top, search for resource group and click "Resource groups" from the list of results.

Installing HyperV and Windows Server 2019

In an enterprise network, servers can be installed on a hypervisor. This allows for easier deployment and the ability to create snapshots. In our lab, we will be using Hyper-V to host Windows Server 2019 and additional Windows 10 machines.
